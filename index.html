Criação do Kotlin

Kotlin foi criada pela JetBrains e lancada em 2011 com o objetivo de ser uma linguagem de programação concisa e segura para o desenvolvimento web, mobile e desktop, com um amplo conjunto de ferramentas e IDEs fáceis de utilizar.

Algumas características:
  • Roda sobre a JVM.
  • Linguagem é baseada em O.O., mesmo os tipos primitivos.
  • Lancada para jetbrains em 2011.
  • Baseada em 4 linguagens Java, scala, groovy, C#.

Kotlin e o Byte Code.

O código de um programa de computador escrito na linguagem Java é compilado para uma forma intermediária de código denominada bytecode, que é interpretada pelas Máquinas Virtuais Java (JVMs). E Kotlin gera o mesmo tipo de ByteCode que Java!

Váriaveis.

Em Kotlin existe dois tipos de váriaveis

Tipo de Variavel mutável.

- Podemos usar sem o tipo:
var name = "Joao"

- Podemos usar com o tipo:
var name: String = "Joao"

Tipo de váriavel mutável - considerada read-only (referência imutável)

val é simular ao final do java ou const do js.

- Podemos user sem o tipo:
val name = "Joao"

- Podemos usar com o tipo:
val name: String = "Joao"

(nesse caso nao podemos mudar o valor da váriavel
// Val cannot be reassigned )

Inicialização de Váriaveis.

Em Kotlin declaramos as variáveis informando as palavras reservadas val ou var, o nome da variável, o tipo e o valor. Podemos omitir o tipo da variável, assim o tipo é inferido pela atribuição de um valor. Portanto nesse caso é obrigatoriamente necessário inicializar a variável.

Se passarmos o tipo, não é necessário inicializar.

Exemplo: 

	val name = "Joao" //Obrigatório inicializar
								

	var name: String; //Não Obrigatório inicializar

Null Safe.

Em kotlin existe um mecanismo de segurança para não apresantar o Famoso NPE. Isso quer dizer que por padráo atritubos não aceita valores nulos, a não ser que forçamos isso.

var str: String = null
//error: null can not be a value of a non-null type String
Operação (?).

Quando precisamos de que nosso objeto seja nulo podemos usar o operador (?), ele vai nos dar a opção de nulo (null) de forma segura

var str: String = null
(?) -operador de chamada segura

			
var str: String? = null //temos que forçar o nulo


str?.lenght //Verifica se a String é nula -> sempre vai retornar um nulo (null) e não uma Exception
Níveis.

Podemos checar vãrios niveis

val logradouro = cliente?.empresa?.endereco?.logradouro


println(logradouro)
Null Exception.

Em casos que precisamos trata a Exception, poderiamos usa o operado !!.

str!!.lenght // Ignora o mecanismo de seguranca do Kotlin para NPE //irá lancar um nullPointException, para ser tratada
Operador Elvis “?:”.

Este operador válida se um objeto é nulo e provê uma forma de devolver um valor default.

val cliente: Cliente? = null
val logradouro = cliente?.empresa?.endereco?.logradouro ?: "Av. Joao naves"
Cast explícito seguro: “as?”.

Usamos o operador as para fazer cast de um tipo de dados para outro tipo e caso não seja possível fazer essa operação é lançada uma Exception:.

val nome: Any? = null
val stringValue = nome as String
// null cannot be cast to non-null type kotlin.String

			
Para fazer o cast de forma segura é necessário usar o operador as? para não acontecer a Exception:

	
val nome: Any? = null
val stringValue = nome as? String
Let

Usado para executar um bloco de código a partir de um objeto de forma segura:.

val nome:String? = null
nome.let { 
	println("nome is $nome")
}
// nome is null
Let

Em conjunto com operador de chamada segura podemos definir que o bloco do let apenas seja executado se o objeto não for nulo:.

val nome:String? = null
nome?.let { 
	println("nome is $nome")
}
// nome é nulo, então não executa o println()
lateinit

É um modificador que permite que uma propriedade var não nula possa ser inicializada depois de sua declaração. Caso não tenha sido inicializada somos avisados com a mensagem:

>>lateinit property name has not been initialized <<

Isso ai nos ajudar a precaver um NPE.

lateinit sem Inicialização
lateinit var name: Any

fun main(args: Array) {
    println(name)
    // lateinit property name has not been initialized
}
lateinit com Inicialização antes de usar

lateinit var name: Any

fun main(args: Array) {
	name = "Joao"
	println(name)
	// name
}
